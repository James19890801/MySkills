# Skill 路由器 — 意图识别与精准匹配

## 你的核心职责

用户说了一个需求，但你不确定该用哪个 skill，或者可能命中多个 skill。
**你的任务：先识别意图，再精准路由，而不是猜测后直接执行。**

## 第二步：意图匹配决策树

```
用户说了需求
    │
    ▼
能唯一命中一个 Skill？
    ├── YES → 直接调用，无需询问
    └── NO（命中2个以上，或完全不确定）
            │
            ▼
        展示候选列表，让用户选择（见第三步）
```

---

## 第三步：命中多个时，展示选项让用户选

**格式模板（简洁，不废话）：**

```
我理解您想要「[用户需求概括]」，以下 skill 都能做，请选择：

1️⃣ [Skill名称] — [一句话说明它能做什么，输出什么]
2️⃣ [Skill名称] — [一句话说明它能做什么，输出什么]
3️⃣ [Skill名称] — [一句话说明它能做什么，输出什么]

您选哪个？（直接说数字即可）
```

**规则：**
- 最多展示 4 个选项，超过 4 个则先筛选最相关的
- 每个选项描述不超过 20 字
- 不解释原理，不废话，直接让用户选
- 用户选完后立即调用对应 skill

## 执行原则

1. **唯一命中 → 直接执行**，不用问用户
2. **多个命中 → 展示选项，让用户选**，绝不自行猜测
3. **完全不确定 → 只问一句话**："您具体想要什么样的输出？是文档/图表/网页/其他？"
4. **禁止**：执行完了才发现用错 skill，导致返工
